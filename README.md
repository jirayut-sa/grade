# ระบบบันทึกคะแนนนักเรียน - โรงเรียนบ้านโคกประดู่

ระบบหน้าบ้าน (Frontend) เป็นไฟล์ **index.html ไฟล์เดียว** ใช้งานผ่าน GitHub Pages ได้ทันที
ส่วนฐานข้อมูลใช้ **Google Sheets** และ API เชื่อมต่อผ่าน **Google Apps Script (Code.gs)**
(บัญชี Google Sheets/Apps Script ฟรี ไม่มีค่าใช้จ่าย)

> ทำไมต้องมี 2 ไฟล์ทั้งที่ขอ "ไฟล์เดียว"? — index.html คือระบบหน้าบ้านทั้งหมด (UI/ระบบ/ตรรกะ)
> อยู่ในไฟล์เดียวจริง ๆ ส่วน Code.gs คือ "ตัวฐานข้อมูล" ที่ต้องรันอยู่บนเซิร์ฟเวอร์ของ Google เพื่อให้
> เขียน/อ่าน Google Sheets ได้อย่างปลอดภัย (เบราว์เซอร์ล้วน ๆ เขียน Google Sheets ตรง ๆ ไม่ได้)
> วิธีนี้เป็นวิธีมาตรฐานสำหรับทำเว็บแอปที่ใช้ Google Sheets เป็นฐานข้อมูลฟรี

## ขั้นตอนที่ 1: สร้างฐานข้อมูล Google Sheets + API

1. ไปที่ [sheets.google.com](https://sheets.google.com) สร้างสเปรดชีตใหม่ ตั้งชื่อ เช่น `DB-คะแนนบ้านโคกประดู่`
2. เมนู **ส่วนขยาย (Extensions) > Apps Script**
3. ลบโค้ดตัวอย่างเดิมทั้งหมด แล้วคัดลอกโค้ดทั้งหมดจากไฟล์ **Code.gs** ที่แนบมาไปวาง
4. บันทึกโปรเจกต์ (ตั้งชื่อ เช่น `ScoreAPI`)
5. ที่แถบเครื่องมือด้านบน เลือกฟังก์ชัน `setup` แล้วกด **Run (▶)**
   - ครั้งแรกจะขึ้นขอสิทธิ์การใช้งาน ให้กด **อนุญาต (Allow)** ทั้งหมด
   - ระบบจะสร้างชีตย่อย (Subjects, Classes, Students, Enrollments, Scores, Users, Sessions) ให้อัตโนมัติ
   - พร้อมสร้างบัญชีผู้ดูแลระบบเริ่มต้น: **username: `admin` / password: `admin1234`**
6. กด **Deploy (ปุ่มสีน้ำเงินมุมขวาบน) > New deployment**
   - ประเภท (Select type): เลือก **Web app**
   - Execute as: **Me (อีเมลของคุณ)**
   - Who has access: **Anyone**
   - กด **Deploy** แล้วกด **Authorize access** อนุญาตสิทธิ์อีกครั้ง
7. คัดลอกลิงก์ **Web app URL** ที่ได้ (จะขึ้นต้นด้วย `https://script.google.com/macros/s/.../exec`) เก็บไว้ใช้ในขั้นตอนถัดไป

> ทุกครั้งที่แก้โค้ด Code.gs ต้องกด **Deploy > Manage deployments > แก้ไข (ไอคอนดินสอ) > New version > Deploy** ใหม่ ลิงก์เดิมจะยังใช้ได้

## ขั้นตอนที่ 2: ตั้งค่าไฟล์ index.html

เปิดไฟล์ `index.html` ด้วยโปรแกรมแก้ไขข้อความ ค้นหาบรรทัด:

```js
const API_URL = "PASTE_YOUR_APPS_SCRIPT_WEB_APP_URL_HERE";
```

แก้เป็นลิงก์ Web app URL ที่คัดลอกมาจากขั้นตอนที่ 1 เช่น:

```js
const API_URL = "https://script.google.com/macros/s/AKfycb.../exec";
```

บันทึกไฟล์

## ขั้นตอนที่ 3: นำขึ้น GitHub Pages

1. สร้าง repository ใหม่บน GitHub (public)
2. อัปโหลดไฟล์ `index.html` (และ `logo.png`, `manifest.json` ถ้ามี) ขึ้น repository
3. ไปที่ **Settings > Pages**
4. เลือก Source: **Deploy from a branch**, Branch: **main**, Folder: **/(root)**
5. กด Save รอสักครู่ ระบบจะให้ลิงก์เว็บไซต์ เช่น `https://ชื่อผู้ใช้.github.io/ชื่อ-repo/`
6. เปิดลิงก์นั้น จะเจอหน้าล็อกอินของระบบ

**กรุณาสร้างบัญชีจริงและลบ/เปลี่ยนรหัสผ่านบัญชี admin เริ่มต้นทันทีหลังติดตั้งเสร็จ** (ไปที่แท็บ "ผู้ใช้งาน" เพื่อเพิ่มผู้ใช้ หรือใช้ฟังก์ชันเปลี่ยนรหัสผ่านในระบบ)

### บทบาทผู้ใช้งาน

| บทบาท | สิทธิ์ |
|---|---|
| **Admin** | จัดการรายวิชา/ชั้นเรียน/นักเรียน/การลงทะเบียน/ผู้ใช้งานทั้งหมด เห็นข้อมูลทุกวิชาทุกห้อง |
| **ครูประจำชั้น** | จัดการ/นำเข้ารายชื่อนักเรียนเฉพาะชั้นของตนเอง เห็นคะแนนทุกวิชาของนักเรียนในชั้นตน บันทึกคะแนนแทนได้ |
| **ครูผู้สอน** | บันทึกคะแนน/ดูรายงาน/ดู AI วิเคราะห์ เฉพาะวิชาที่ตนได้รับมอบหมายเท่านั้น |

การกำหนดว่าครูประจำชั้นดูแลชั้นไหน หรือครูผู้สอนสอนวิชาอะไร ให้ Admin ตั้งค่าตอนสร้างบัญชีในแท็บ "ผู้ใช้งาน"

## การนำเข้ารายชื่อจาก Excel

ไปที่แท็บ **จัดการข้อมูล > นักเรียน > นำเข้า Excel**
ไฟล์ที่รองรับ: `.xlsx`, `.xls`, `.csv`
กติกา: **คอลัมน์ A ของไฟล์ต้องเป็นชื่อ-สกุลนักเรียน** (มีหรือไม่มีแถวหัวตารางก็ได้ ระบบจะพยายามตัดหัวตารางออกให้อัตโนมัติถ้าตรวจพบคำว่า "ชื่อ" หรือ "name")

## AI วิเคราะห์ผลการเรียน

ระบบมีโหมดวิเคราะห์ 2 แบบ:

1. **วิเคราะห์เชิงสถิติอัตโนมัติ (ใช้งานได้ทันที ไม่ต้องตั้งค่าอะไรเพิ่ม)** — คำนวณค่าเฉลี่ย ส่วนเบี่ยงเบนมาตรฐาน แนวโน้มคะแนนรายรอบ รายชื่อกลุ่มเสี่ยง/กลุ่มเก่ง แล้วสรุปเป็นข้อความให้ครูอ่านทันที
2. **วิเคราะห์ด้วย AI จริง (ทางเลือกเสริม)** — เชื่อมกับ Gemini API (มีโควตาใช้งานฟรี) เพื่อให้ AI เขียนสรุปและคำแนะนำเชิงลึกกว่าเดิม
   - ขอ API Key ฟรีที่ [aistudio.google.com/apikey](https://aistudio.google.com/apikey)
   - กลับไปที่ Apps Script โปรเจกต์ `ScoreAPI` > ไอคอนฟันเฟือง **Project Settings** > **Script Properties**
   - เพิ่มค่า Property name: `GEMINI_API_KEY` และ Value: วางคีย์ที่ได้มา แล้วบันทึก
   - ไม่ต้องแก้ไฟล์ index.html เพิ่มเติม ระบบจะตรวจพบและสลับไปใช้ AI จริงให้อัตโนมัติ

ถ้าไม่ตั้งค่า Gemini API Key ไว้ ระบบจะใช้การวิเคราะห์เชิงสถิติ (ข้อ 1) แทนเสมอ ใช้งานได้ครบถ้วนโดยไม่มีค่าใช้จ่าย

## โครงสร้างข้อมูลใน Google Sheets

| ชีต | คอลัมน์ |
|---|---|
| Subjects | SubjectID, SubjectName, TeacherUsernames |
| Classes | ClassID, ClassName, HomeroomUsername |
| Students | StudentID, FullName, ClassID |
| Enrollments | SubjectID, ClassID, StudentID |
| Scores | ScoreID, SubjectID, ClassID, StudentID, Round, ScoreValue, Timestamp |
| Users | Username, PasswordHash, Salt, Role, FullName, ClassID, SubjectIDs |
| Sessions | Token, Username, ExpiresAt |

สามารถเปิดชีต Users/Sessions/Scores ฯลฯ ดูข้อมูลดิบได้โดยตรงจาก Google Sheets เพื่อสำรองข้อมูลหรือทำรายงานเพิ่มเติมเองได้ทุกเมื่อ (ไฟล์อยู่ใน Google Drive ของคุณเอง จึงถือว่าเชื่อมกับ Google Drive แล้วในตัว)

## แก้ปัญหาเบื้องต้น

- **เข้าสู่ระบบไม่ได้ / ขึ้น error เชื่อมต่อ**: ตรวจสอบว่า API_URL ในไฟล์ index.html ถูกต้อง และ Deploy เป็น "Anyone" แล้ว
- **แก้โค้ด Code.gs แล้วไม่มีผล**: ต้อง Deploy เวอร์ชันใหม่ทุกครั้ง (Manage deployments > New version)
- **ลืมรหัสผ่าน admin**: เปิด Google Sheet ไปที่ชีต `Users` แล้วลบแถวของ admin ออก จากนั้นกลับไป Apps Script รันฟังก์ชัน `setup` อีกครั้งเพื่อสร้างบัญชี admin ใหม่ (admin/admin1234)
